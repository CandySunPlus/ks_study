# Proposal: auto-advance-on-correct

## 为什么

答对选择题后需要手动点击才能进入下一题，增加了不必要的操作步骤，降低了刷题效率。用户期望在答对后能快速进入下一题，减少等待时间。

## 变更内容

在练习模式和复习模式下，选择题答对后自动进入下一题：

- **答对**：300ms 后自动跳转
- **答错**：停留，等待用户手动确认
- **适用题型**：单选题（type 0）和多选题（type 1）
- **不适用**：简答题（type 3）和背诵模式

### 行为细节

| 场景 | 当前行为 | 变更后行为 |
|------|---------|-----------|
| 练习模式 + 答对 | 停留 + 手动点下一题 | 300ms 后自动跳转 |
| 练习模式 + 答错 | 停留 + 手动点下一题 | 停留 + 手动点下一题（不变） |
| 复习模式 + 答对 | 停留 + 手动点下一题 | 300ms 后自动跳转 |
| 复习模式 + 答错 | 停留 + 手动点下一题 | 停留 + 手动点下一题（不变） |
| 简答题 | 点击"显示答案" | 不变 |
| 背诵模式 | 阅读式显示 | 不变 |

## 功能 (Capabilities)

### 新增功能

- `auto-advance`: 答对后自动进入下一题

## 影响

- **代码**：`app.js` 的 `submitAnswer()` 函数
- **行为**：练习模式和复习模式的答题流程
