## 新增需求

### 需求:支持随机题目顺序
系统必须支持随机打乱题目顺序，根据用户选择的练习模式决定题目呈现顺序。

#### 场景:顺序模式加载题目
- **当** 用户选择"顺序练习"模式进入练习
- **那么** 系统按 JSON 文件中的原始顺序呈现题目
- **并且** 题目导航面板中的题号对应原始位置

#### 场景:随机模式打乱题目
- **当** 用户选择"随机练习"模式进入练习
- **那么** 系统使用 Fisher-Yates shuffle 算法随机打乱题目顺序
- **并且** 题目导航面板中的题号对应随机后的位置

#### 场景:每次进入随机模式都重新打乱
- **当** 用户选择"随机练习"模式进入练习
- **并且** 用户之前已经以随机模式进入过该科目
- **那么** 系统重新生成随机顺序（不复用之前的随机顺序）

#### 场景:随机模式不保存随机种子
- **当** 用户以随机模式进入练习
- **那么** 系统不将随机顺序保存到 localStorage 或 sessionStorage
- **并且** 下次进入时会生成不同的随机顺序

### 需求:显示当前练习模式
系统必须在练习页面清晰显示当前使用的练习模式。

#### 场景:顺序模式显示
- **当** 用户以顺序模式进入练习
- **那么** 练习页面标题显示为"📝 练习模式"
- **并且** 不显示额外的模式标识

#### 场景:随机模式显示
- **当** 用户以随机模式进入练习
- **那么** 练习页面标题显示为"📝 练习模式 🔀"
- **并且** 🔀 emoji 清晰标识当前为随机模式

### 需求:保持现有功能不受影响
随机题目顺序禁止影响答题状态追踪、错题记录等现有功能。

#### 场景:答题状态基于 questionId
- **当** 用户在随机模式下答题
- **那么** 系统使用 questionId 而非题目位置索引追踪答题状态
- **并且** 答题状态正确保存到 localStorage

#### 场景:错题记录不受影响
- **当** 用户在随机模式下答错题目
- **那么** 系统正确记录该题目的 questionId 到错题列表
- **并且** 复习模式能正确加载该错题

#### 场景:导航面板状态正确显示
- **当** 用户在随机模式下答题
- **那么** 导航面板中的题号状态（✓✗）正确反映答题结果
- **并且** 点击导航面板题号能正确跳转到对应题目

#### 场景:答案选择恢复功能正常
- **当** 用户在随机模式下答题后返回已答题目
- **那么** 系统正确恢复用户之前的答案选择
- **并且** 显示之前的答题结果和反馈

## 修改需求

### 需求:Track answer status
系统必须追踪哪些题目已被作答以及答题正确与否，并在会话间持久化此信息。在随机模式下，系统必须使用 questionId 而非题目索引进行追踪。

#### 场景:Mark question as answered
- **当** 用户提交答案
- **那么** 系统在会话状态和持久化存储中标记该题目为已完成
- **并且** 系统使用 questionId 作为存储键，而非题目位置索引

#### 场景:Allow re-answering
- **当** 用户导航回之前已答的题目
- **那么** 系统允许重新选择和提交答案
- **并且** 在随机模式下，系统能正确识别并恢复该题目的答题状态

#### 场景:Persist answer status across sessions
- **当** 用户答题后关闭浏览器
- **那么** 系统将答题状态存储到 localStorage
- **并且** 存储使用 questionId 为键，确保随机顺序不影响状态追踪

#### 场景:Restore answer status on session start
- **当** 用户在之前答题后开始新的练习会话
- **那么** 系统从 localStorage 加载答题状态并初始化会话状态
- **并且** 在随机模式下，系统根据 questionId 正确恢复每个题目的状态

#### 场景:Update persistent storage on re-answer
- **当** 用户重新作答之前已答的题目
- **那么** 系统同时更新会话状态和持久化存储中的新状态
- **并且** 更新操作基于 questionId，不受题目顺序影响
