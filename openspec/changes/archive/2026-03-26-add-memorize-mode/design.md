## 上下文

本项目是一个基于纯 JavaScript 的刷题工具，当前已实现：
- **练习模式**：用户选择答案后提交，系统反馈正确/错误
- **复习模式**：只显示错题，进行针对性练习
- **简答题（type 3）**：需要点击"显示答案"按钮才能看到答案

题目数据存储在 JSON 文件中，包含 `title`、`answerOptions`、`answer`、`analysis` 等字段。

## 目标 / 非目标

**目标：**
- 实现背题模式，允许用户直接浏览题目及其正确答案和解析
- 在练习/复习模式提交答案后显示题目解析
- 在背题模式下对错题进行视觉标识
- 保持代码结构清晰，复用现有渲染逻辑

**非目标：**
- 不修改现有数据存储格式
- 不记录背题模式的浏览历史
- 不添加复杂的背题计划或进度追踪功能
- 不改写为框架实现（保持原生 JS）

## 决策

### 1. 背题模式作为独立状态标识
**决策**：使用 `isMemorizeMode: true` 状态标识，而非创建全新视图组件。

**理由**：
- 背题模式与练习模式的 UI 结构高度相似（题目、选项、导航）
- 复用现有 `practice-view` 容器，减少 DOM 复杂度
- 只需在渲染逻辑中区分模式行为

**替代方案**：创建独立的 `memorize-view` DOM 容器。否决原因：90% 结构重复，维护成本高。

### 2. 选择题答案显示复用 feedback 样式
**决策**：背题模式下直接应用类似 `showFeedback` 的高亮样式，但不禁用选项点击。

**理由**：
- 保持视觉一致性（绿色 = 正确）
- 选项无需禁用（用户知道这是背题模式，不会误操作）

**具体实现**：
```javascript
// 背题模式直接为正确选项添加 'correct' 类
option.classList.add('correct');
// 单选样式：显示为已选中
input.checked = true;
```

### 3. 解析显示区域复用 feedback 容器
**决策**：将解析内容追加到现有的 `feedback` 元素中。

**理由**：
- feedback 区域已经用于显示正确/错误状态
- 解析作为补充信息，放在 feedback 区域符合用户预期
- 无需新增 DOM 元素

### 4. 错题标识采用标题前缀 + 导航面板高亮
**决策**：
- 题目标题前加 ❌ 符号（简单直观）
- 导航面板错题格子使用 `incorrect` CSS 类（红色背景，复用现有样式）

**理由**：
- ❌ 符号跨文化通用，无需额外样式
- 复用 `incorrect` 类保持设计一致性

### 5. 背题模式反馈消息隐藏
**决策**：背题模式下不显示"✓ 正确答案"提示框，只显示解析内容（如果有）。

**理由**：
- 背题模式的目标是浏览和记忆，不需要"答题正确"的反馈
- 正确答案已经通过选项高亮直接展示
- 只保留解析内容，避免信息冗余

### 6. showFeedback 消息元素可见性修复
**决策**：在 `showFeedback` 函数中显式设置 `messageEl.style.display = 'block'`。

**理由**：
- 背题模式下隐藏了 `feedback-message` 元素
- 练习/复习模式调用 `showFeedback` 时必须重新显示消息元素
- 避免跨模式状态污染

## 风险 / 权衡

| 风险 | 缓解措施 |
|------|----------|
| 用户混淆练习模式和背题模式 | 首页按钮明确标注"背题模式"，页面顶部显示"背题"标识 |
| 解析内容可能很长，影响布局 | 解析区域使用 `max-height` + `overflow-y: auto`，或默认折叠 |
| 简答题答案很长，直接显示占用空间 | 保持直接显示，用户可通过 N/P 快速切换 |

## 迁移计划

无需迁移，此变更为纯功能添加，不影响：
- 现有用户数据（localStorage 格式不变）
- 现有练习/复习模式行为
- 题目 JSON 数据格式
