## ADDED Requirements

### 需求:用户可以从首页进入背题模式
系统必须在首页 Dashboard 提供背题模式的入口，允许用户选择科目进入背题模式。

#### 场景:用户点击背题模式按钮
- **当** 用户在首页点击"背题模式"按钮
- **那么** 系统显示科目选择界面或直接进入该科目的背题模式

### 需求:背题模式下直接显示选择题正确答案
系统在背题模式下必须直接显示选择题的正确答案，无需用户进行选择和提交操作。

#### 场景:选择题直接显示答案
- **当** 用户进入一道选择题的背题模式
- **那么** 系统立即以高亮形式显示正确选项
- **并且** 选项不可点击或选择

### 需求:背题模式下直接显示简答题答案
系统在背题模式下必须直接显示简答题的答案，不需要用户点击任何按钮。

#### 场景:简答题直接显示答案
- **当** 用户进入一道简答题的背题模式
- **那么** 系统立即显示答案内容
- **并且** 不显示"显示答案"按钮

### 需求:背题模式下显示题目解析
如果题目数据包含解析内容，系统在背题模式下必须直接显示该解析。

#### 场景:有解析的题目直接显示解析
- **当** 用户进入一道包含解析字段的题目
- **那么** 系统直接显示解析内容

#### 场景:无解析的题目不显示解析区域
- **当** 用户进入一道不包含解析字段的题目
- **那么** 系统不显示解析区域

### 需求:背题模式下标识错题
系统在背题模式下必须在错题的标题处显示错题标识。

#### 场景:错题显示标识
- **当** 用户进入一道错题的背题模式
- **那么** 系统在题目标题处显示错题标识（如 ❌）

### 需求:背题模式下导航面板标识错题
系统在背题模式的导航面板中必须以特殊样式（如红色）标识错题格子。

#### 场景:导航面板错题格子高亮
- **当** 用户在背题模式下查看导航面板
- **那么** 错题对应的格子以红色背景显示

### 需求:背题模式不记录答题统计
系统在背题模式下不得记录任何答题统计信息，包括正确率、完成数等。

#### 场景:浏览背题模式不影响统计
- **当** 用户在背题模式下浏览题目
- **那么** 系统的答题统计数据保持不变

### 需求:背题模式支持键盘快捷键
系统在背题模式下必须支持键盘快捷键进行导航。

#### 场景:使用P键返回上一题
- **当** 用户按下 P 键
- **那么** 系统显示上一道题目

#### 场景:使用N键进入下一题
- **当** 用户按下 N 键
- **那么** 系统显示下一道题目

#### 场景:使用Esc键退出背题模式
- **当** 用户按下 Esc 键
- **那么** 系统返回首页
